# PostgreSQL backup and restore

Production backups are generated by `ops/postgres_backup.sh`. Store a copy outside the VM boot disk.

## Verify a backup

```bash
gzip -t /var/lib/arbitrage/backups/arbitrage-YYYYMMDDTHHMMSSZ.sql.gz
```

## Restore drill

Never restore over the production database. Create an isolated database, restore it, run migrations, and compare row counts.

```bash
createdb arbitrage_restore_test
gzip -dc /var/lib/arbitrage/backups/arbitrage-YYYYMMDDTHHMMSSZ.sql.gz \
  | psql --set ON_ERROR_STOP=1 arbitrage_restore_test
DATABASE_URL=postgresql+asyncpg://arbitrage@127.0.0.1/arbitrage_restore_test alembic upgrade head
psql arbitrage_restore_test -c '\\dt'
dropdb arbitrage_restore_test
```

Record the backup name, SHA-256, restore duration, migration revision, and operator in the deployment log.
